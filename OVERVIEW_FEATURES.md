# ภาพรวมฟีเจอร์ทั้งหมดของระบบ (Project Features Overview)

สรุปความสามารถปัจจุบันของระบบจัดการร้านซ่อมรถจักรยานยนต์ (Motorcycle Service Management) ทั้งหมด

---

## 1. ระบบจัดการผู้ใช้และการเข้าถึง (Authentication & Access Control)

- **ระบบ Login/Register:** สมัครสมาชิกและเข้าสู่ระบบผ่าน Supabase Auth
- **Role-Based Access (RBAC):** แยกสิทธิ์การใช้งานออกเป็น 3 ระดับ:
  - **Customer (ลูกค้า):** จัดการรถของตัวเอง, จองคิว, ติดตามงานซ่อม
  - **Staff (เจ้าหน้าที่/ช่าง):** จัดการคิวจอง, ทำใบเสนอราคา, บันทึกการซ่อม, จัดการสต็อก
  - **Admin (ผู้ดูแลระบบ):** จัดการพนักงานและดูภาพรวมระบบทั้งหมด
- **Profile Management:** จัดการข้อมูลส่วนตัว เบอร์โทรศัพท์ และข้อมูลพื้นฐาน

## 2. ฟีเจอร์สำหรับลูกค้า (Customer Experience)

- **ระบบจองคิวซ่อม (Booking):** เลือกวัน/เวลา และระบุอาการเบื้องต้นผ่านระบบปฏิทิน
- **จัดการรถจักรยานยนต์ (Motorcycle Management):**
  - ลงทะเบียนรถ พร้อมแนบรูปภาพรถจริง (Image Upload)
  - แสดงผลการ์ดรถแบบพรีเมียม (DaisyUI Cards)
- **ติดตามสถานะงานซ่อม (Repair Tracking):**
  - ดูความคืบหน้าแบบ Real-time (Progress Bar)
  - กดยืนยันราคาประเมินก่อนเริ่มซ่อม
  - กดยกเลิกงานซ่อม (หากยังไม่เริ่มดำเนินการ)
- **ระบบคะแนนสะสม (Loyalty Points):** สะสมแต้มจากการใช้บริการเพื่อใช้เป็นส่วนลดในอนาคต
- **ประวัติการชำระเงิน (Payments):** ดูประวัติและสถานะการจ่ายเงินของแต่ละงานซ่อม

## 3. ฟีเจอร์สำหรับเจ้าหน้าที่และช่าง (Staff & Technician Ops)

- **การจัดการรายการจอง (Dashboard):** ดูรายการจองใหม่ และจัดการสถานะการรับรถ
- **การประเมินราคา (Estimates):** บันทึกรายละเอียดอาการและราคาประเมินเบื้องต้นส่งให้ลูกค้า
- **การจัดการงานซ่อม (Repair Management):**
  - บันทึกอะไหล่ที่ใช้ (เชื่อมโยงกับสต็อก) และค่าแรง
  - อัปเดตเปอร์เซ็นต์ความคืบหน้า (Progress)
- **ระบบคลังอะไหล่ (Inventory Management):**
  - จัดการสต็อกอะไหล่ พร้อมแนบรูปภาพสินค้า (Image Upload)
  - ระบบแจ้งเตือนสินค้าใกล้หมด (Low Stock Alert)
  - คำนวณมูลค่ารวมของคลังอะไหล่โดยอัตโนมัติ

## 4. ฟีเจอร์สำหรับผู้ดูแลระบบ (Admin Control)

- **User Management:** จัดการและบริหารรายชื่อผู้ใช้และพนักงานในระบบ
- **Analytics Dashboard:** ดูสถิติรายได้, งานซ่อมที่ค้างอยู่ และภาพรวมการจอง

## 5. ระบบโครงสร้างพื้นฐาน (Core Systems)

- **Supabase Client Side Storage:** จัดเก็บรูปภาพรถและอะไหล่ใน Bucket แยกเฉพาะ
- **Database Transaction:** ระบบความปลอดภัยของข้อมูลที่มั่นใจว่าข้อมูล Booking และ Repair จะสัมพันธ์กันเสมอ
- **Rich UI/UX:** ใช้ DaisyUI และ Tailwind CSS ในการสร้างอินเตอร์เฟซที่ทันสมัย รองรับ Dark Mode และ Responsive (มือถือ/แท็บเล็ต)
- **Internal Logging:** ระบบสรุปความคืบหน้าการพัฒนาลงในไฟล์ markdown พื่อการติดตามงาน
